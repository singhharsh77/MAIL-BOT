## üìã Problem Statement
Customer Success (CS) and Operations teams often spend hours daily manually drafting follow-up emails. While standard templates exist, they require significant manual editing to include specific context from meetings or support tickets. This manual process leads to:
* **Inconsistent Brand Voice:** Variation in tone across different team members.
* **Response Latency:** Time-consuming drafting delays client communication.
* **Operational Risk:** High probability of human error when manually copying technical details like ticket numbers or URLs.

## üí° Solution
The **AI Mail Bot** automates the "context-filling" phase of email drafting. By selecting a base template and providing raw input (such as a meeting transcript or internal notes), the tool uses OpenAI to:
1.  **Extract relevant data** (Action items, names, specific technical issues).
2.  **Synthesize context** into the selected professional template.
3.  **Offer a "Human-in-the-Loop" workflow** via a Rich Text Editor for final refinements before sending.

## üíª Development & Environment
### Why `code.gs` cannot "run" directly in VS Code
This project runs on the **Google Apps Script (GAS)** V8 engine. 
* **Backend Logic:** The `.gs` files utilize Google-exclusive services (`MailApp`, `HtmlService`, `PropertiesService`) that are not present in standard Node.js or browser environments.
* **Execution:** Code must be pushed to Google‚Äôs servers to function.

### Recommended Local Workflow
To develop in VS Code with full version control, use **clasp** (Command Line Apps Script Projects):
1.  **Install Clasp:** `npm install -g @google/clasp`
2.  **Authenticate:** `clasp login`
3.  **Sync:** Use `clasp pull` to get files and `clasp push` to upload local changes to the Google Script engine.

## üîß Setup & Installation

1.  **Create Project:** Open [script.google.com](https://script.google.com) and create a new project.
2.  **Add Files:** * Create a file named `index.html` and paste the provided frontend code.
    * Create a file named `code.gs` and paste the backend logic.
3.  **Configure API Key:**
    * Navigate to **Project Settings** (gear icon).
    * Add a **Script Property**: 
        * Property: `OPENAI_API_KEY`
        * Value: `your_secret_api_key_here`
4.  **Deploy:**
    * Click **Deploy** > **New Deployment** > **Web App**.
    * Set access to "Anyone with Google Account" or your specific organization.
    * Authorize the necessary permissions (Gmail, External Service).
